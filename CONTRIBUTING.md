## Contributing Guidelines

Contributions to the MÃ¶bius-Original repository are welcome and should adhere to the following guidelines to maintain code quality and consistency.

### How to Contribute

1. *Fork the Repository*: Create a personal fork on GitHub.  
2. *Clone the Fork*: Clone your fork to your local machine.  
3. *Create a Branch*: Use a descriptive branch name, such as feature/new-tool-integration or fix/bug-in-proxy-manager.  
4. *Make Changes*: Implement your changes, ensuring they align with the repository's style: strict, official, and well-documented. Follow best practices for the relevant programming languages and ensure compatibility with the platform's unified workflow.  
5. *Test Your Changes*: Verify functionality and ensure no regressions in existing applications.  
6. *Commit Changes*: Use clear, concise commit messages.  
7. *Push to Fork*: Push your branch to your GitHub fork.  
8. *Submit a Pull Request*: Open a pull request against the main branch of the original repository. Include a detailed description of the changes, rationale, and any related issues.

### Code Standards

- Adhere to standard style guidelines for the languages used in the project.  
- Include docstrings or comments for functions, classes, and key logic.  
- Avoid introducing new dependencies unless necessary; justify them in the pull request.  
- Ensure cross-platform compatibility and integration with existing tools.

### Issue Reporting

Before contributing, check existing issues. If reporting a new issue:  
- Use a clear title and detailed description.  
- Include steps to reproduce, expected vs. actual behavior, and environment details.

### Review Process

Pull requests will be reviewed by the maintainer. Feedback may require revisions. Approved changes will be merged into the main branch.

By contributing, you agree that your contributions will be licensed under the repository's license.
